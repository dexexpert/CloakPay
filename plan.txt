1. What the requirements really mean

Cloak is looking for products where public Solana transactions create a real business problem.

Normal Solana payments expose:

sender
receiver
amount
timing
wallet history

Cloak hides sensitive payment details using a shielded pool, client-side ZK proofs, private transfers, batch disbursement, stealth addresses, viewing keys, and private swaps. The official docs describe the TypeScript SDK as the main choice for web apps, browser wallets, and Node.js backends, with methods such as deposit, withdraw, send, and swap.

The judging is:

Category	Weight	What judges want
Integration depth	40%	Cloak must do the real private payment work
Product	30%	Clean UX, working demo, reliable flow, good error states
Real-world use	30%	Clear user, clear pain, privacy must be necessary

So to win, your project should feel like:

“This product cannot work properly without Cloak.”

Not:

“We added a private payment button.”

11. README structure that judges will like

Your README should be direct.

# CloakPay

Private payroll and contractor payments for Solana teams.

## Problem

Solana payments are public by default. For teams paying contractors in USDC, this exposes salaries, recipients, timing, and treasury movement.

## Solution

CloakPay lets an organization run private payroll batches through Cloak. Recipients get paid without exposing payment amounts or destination addresses publicly. Finance teams can still review payroll through viewing keys.

## Who it is for

- Solana startups
- DAOs
- agencies
- teams paying contractors in USDC

## How Cloak is used

Cloak is central to the product:

1. Deposit payroll funds into Cloak shielded balance
2. Execute private batch disbursement
3. Use viewing keys for audit reports

## Architecture

Frontend: Next.js + TypeScript  
Wallet: Solana wallet adapter  
Privacy layer: Cloak SDK  
Database: Supabase/Firebase/local demo store

## Run locally

pnpm install
cp .env.example .env.local
pnpm dev

## Demo flow

1. Connect wallet
2. Upload payroll CSV
3. Shield funds
4. Run private payroll
5. Open audit portal
6. Export report
